# 📌 全项目浓缩成 5 个部分（PPT 主结构）

> 一条主线贯穿:**真实巡检场景 → 据此(1分析场景)(2搭仿真环境)(3定模型架构) → (4闭环)(5下一步)**。
> 五个 4 字标题:**① 巡检场景 ② 真实采数 ③ 仿真环境 ④ 双脑架构 ⑤ 闭环展望**
> 串场:场景(①②)决定环境(③)与架构(④),最后闭环(⑤)。

### ①【巡检场景】项目定位与目标场景
- 解决**机器人巡检**:移动机器人在真实园区/室内外按**任务**行进、识别响应不同目标。
- 两个场景本质特征(后面所有设计的出发点):**任务驱动**(同路不同任务→不同轨迹)、**真实复杂环境**(室内外/长通道/光照多变)。

### ②【真实采数】真实数据采集(场景刻画)
- 2026-06-11 实地采集 **6 段 ~59min、7000+帧**,主感知 **ZED 双目**(+IMU/ROS2)。
- 6 段覆盖巡检多形态(开阔巡行/室内走廊/往返巡逻/定点观察)——**这批数据刻画了真实场景,是下面架构与环境设计的共同依据**(不是"缺了哪些数据"清单)。

### ③【仿真环境】仿真测试环境(从"真实复杂环境"推出的设计)
- **流水线(已打通)**:ZED 图像 →①VGGT-Ω 重建(点云+位姿)→②训练 3DGS(照片级)→③对齐米制世界帧 →④装进 **DISCOVERSE 物理仿真器**(MuJoCo+3DGS,差速机器人+车载相机)→⑤**浏览器 WASD 亲手走**。详见 P5。
- 思路:真实采集 → 重建 → 能跑算法的仿真台,**几何全锚定真实传感器,不造假**。
- ⭐**已打通整条链(最新)**:VGGT-Ω 重建 → 训练 **3DGS** → 装进**真实 DISCOVERSE 物理仿真器**(机器人+车载相机)→ **浏览器 WASD 亲手开机器人走过重建的真实场景**(headless 云端实时渲染)。
- 另有两套 2D 交付(抽象 facade / 标准 ROS 占据图)。**放图/视频**:`outputs/sim_walk_114830.mp4`、VGGT BEV。

### ④【双脑架构】模型架构(从"任务驱动"推出的设计)
- ⭐**架构图(DiffusionDrive 风格,可直接放 PPT)**:`outputs/arch_diagram.png`(脚本 `scripts/draw_arch.py`)。
- 核心 framing(论文创新点):把 **DiffusionDrive**(车端AD扩散轨迹规划器)**迁移到任务驱动巡检机器人**。
- **双层双脑**:慢脑 VLM(Qwen3级)识场景→**离散任务id**;快脑 DiffusionDrive(实时2步去噪)按**任务id条件**生成轨迹。创新点=任务id注入级联解码器,对应场景的"任务驱动"。端侧 Jetson Orin。

### ⑤【闭环展望】闭环与下一步
- **闭环**:真实场景重建 → real2sim → 仿真造训练场+示教 → 训 VLM/DiffusionDrive → 真机 Orin。
- **下一步**:等工控机数据定底盘/接口/Orin;提升重建画质(更密帧VGGT-Ω);跑通 DiffusionDrive 基线接任务条件;把仿真器当训练台。

## P1 封面 / 一句话定位（~20s）
- **标题**：机器人巡检 —— 面向真实场景的任务驱动双层导航框架 + BEV 仿真测试台
- **一句话**：从真实巡检场景出发，设计「VLM 识任务 + DiffusionDrive 出轨迹」的双层框架，并用真实采集数据搭出能测这套框架的仿真环境。
- 口播："这阶段的逻辑很清楚：先看清楚巡检这个场景长什么样，再据此定架构、搭环境。"

## P2 目标场景：机器人巡检（~40s，定调）
- 我们要解决的具体场景：**移动机器人在真实园区/室内外环境里做巡检**——沿走廊/通道行进，**识别并响应不同的目标与区域**（设备、仪表、出入口…），按当前**任务**决定怎么走。
- 这个场景的两个本质特征（后面所有设计都从这里推）：
  1. **任务驱动**：同一条路，巡检任务不同 → 期望轨迹/行为不同（不是单纯"沿路开"）。
  2. **真实环境复杂**：室内外混合、长通道、光照/结构多变 → 需要贴近真实的测试环境，不能凭空造假场景。
- 口播："巡检不是简单沿路走，而是按任务走在复杂真实环境里——这两点决定了我们后面的全部设计。"

- 平台：移动底盘 + **ZED 双目相机**（主感知，出 RGB + 米制点云）+ IMU + ROS2。
- 6 段覆盖了巡检场景的不同形态（建议做成表）：

| 112633 | 7.5 min | 897 | 开阔区域 |

## P4 ⭐ 核心逻辑：从场景 → 两条设计（~50s，全场最关键一页）
> 建议做成一张「场景 → 两个箭头」的图：左边"真实巡检场景/数据"，右分两路。

- **场景特征①「任务驱动」→ 决定模型架构**：
  因为同一环境下不同巡检任务要走不同轨迹 → 我们设计**双层双脑**：上层 VLM 识别场景/任务，下层 DiffusionDrive **按任务条件**生成轨迹。（详见 P6）
- **场景特征②「真实复杂环境」→ 决定仿真环境**：
  因为要在贴近真实的环境里测这套框架、又不能造假场景 → 我们用**这批真实数据重建场景**，搭出 BEV/俯视的仿真测试台。（详见 P5）
- 口播："一句话——因为巡检是'任务驱动'，我们才设计了任务条件的双层架构；因为环境是'真实复杂'，我们才坚持用真实数据重建仿真环境。这是我们所有工作的出发点。"

## P5 设计B：用真实数据搭仿真测试环境（~60s，放视频，重点页）
> ⭐ **整条 real2sim 链已打通,机器人能在重建的真实场景里走。** 建议做成一张"五步流水线"图。

**仿真搭建流水线(真实采集 → 能走进去的仿真器):**
1. **采集** ZED 双目图像(真实巡检场景,session 114830 室内走廊)。
2. **VGGT-Ω 前馈重建**:图像 → 稠密点云 + 64 个相机位姿(`run_vggto.py`)。替代了早期 COLMAP,更快更干净。
3. **训练 3DGS**:用 VGGT 的位姿+点云+原图监督,gsplat 训出照片级高斯场景 `gs_vggto_114830.ply`(63.5万高斯,A100 ~5min,`train_3dgs_vggt.py`)。
4. **对齐到仿真世界帧**(`align_gs_world.py`):重力对齐(z-up)+ 米制标定(相机离地~1.2m锚定)+ 地面落 z=0,烘焙成世界帧资产。
5. **装进真实物理仿真器**:DISCOVERSE(MuJoCo 物理 + 3DGS 渲染),放**差速机器人 + 车载相机**,3DGS 场景作背景,**headless EGL 离屏渲染**(`sim_walk_*.py`)。

**仿真器栈**:**DISCOVERSE**(清华 DISCOVERSE 团队,3DGS+MuJoCo 的 real2sim2real 机器人仿真框架)+ `gaussian_renderer` 实时高斯渲染 + MuJoCo 物理。本机无显示 → EGL 离屏 + **浏览器流式交互**。

**产出(放 PPT)**:
- 🖼️ 静态配图(PPT 视频播不了时用):`outputs/sim_walk_114830_frame.png`(仿真器里机器人第一视角穿行走廊)。
- 🎬 `outputs/sim_walk_114830.mp4`:机器人在重建的真实走廊里穿行(仿真器渲染)。
- 🕹️ **浏览器实时漫游**:laptop 浏览器 WASD **亲手开机器人走过重建场景**(云端 A100 实时渲染,`sim_walk_server.py`)。
- 备选 2D 交付:抽象 facade 2D 仿真 / 标准 ROS 占据图(.pgm,nav2/Stage 直接加载)。

- 口播："这条链我们已经打通——真实采集的图像,经 VGGT-Ω 重建、训成 3DGS、对齐到米制世界,装进 DISCOVERSE 物理仿真器。现在能在浏览器里用 WASD 亲手开着机器人,走过我们重建的真实场景。几何全部锚定真实传感器,不是手画的。"

## P6 设计A：任务驱动的双层 / 双脑架构（~50s，放架构图）
- **核心 framing（论文创新点）**：把 **DiffusionDrive**（车端自动驾驶的扩散轨迹规划器）**迁移到任务驱动的巡检机器人**。
- **慢脑** VLM（Qwen3-3B 级，~5s/次）：识场景 → 输出**离散任务 id**（只给"场景区分度"，不建米制地图）。
- **快脑** DiffusionDrive（实时，2 步去噪）：BEV+ego+**任务 id** → 条件生成多模轨迹 → top-1。
- **创新点**：把 VLM 任务 id 注入级联解码器的 cross-attention → 从"沿路开"变成"**按任务选轨迹模式**"——**正好对应 P2 的"任务驱动"特征**。
- 双脑掩盖延迟；端侧部署 **Jetson Orin**（VLM 量化/小模型或 API）。
- 口播："架构直接回应'任务驱动'：上面 VLM 识任务，下面 DiffusionDrive 按任务条件出轨迹，创新点就是这个任务条件注入。"
