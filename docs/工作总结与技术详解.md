## 2. 数据是什么（怎么探明的）

### 2.1 探明方法：先盘点，不要急着写代码

拿到一个陌生数据集，第一步永远是**摸清结构**，而不是猜。用的命令套路：

```bash
# 看 tar 里有什么，不用解包
tar -tf data.tar | awk -F/ '{print $1}' | sort -u      # 顶层目录
tar -tf data.tar | grep -E '\.png$|\.jpg$' | ...        # 按类型统计

# 看一个文件的真实格式（关键！不要靠文件名猜）
file sample.jpg            # 文件头识别
python3 -c "import cv2; print(cv2.imread('x',-1).shape, .dtype)"
```

**踩的坑（值得记）**：第一次传过来的 `.tar.xz` 是损坏的，因为 **xz 还在压缩、传了个半成品**。
判断完整性的命令：

```bash
xz -t file.tar.xz && echo OK      # 测试完整性
stat -c '%s' file                 # 隔几秒看两次，大小变化=还在写
```

教训：**压缩/传输大文件，一定等命令真正结束（提示符回来）再动，并校验完整性。**
另外 JPG/rosbag 已经是压缩格式，**别再用 xz/zip 压**（压不动还慢），纯 `tar` 打包即可。

## 3. 关键技术详解

### 3.1 读 rosbag2，不用装 ROS —— 因为 db3 就是 SQLite

**原理**：ROS2 的 `.db3` 录包本质是个 SQLite 数据库，固定两张表：
- `topics(id, name, type, ...)` —— 话题清单
- `messages(topic_id, timestamp, data)` —— 每条消息，`data` 是二进制 blob

所以**纯 sqlite3 就能读话题清单**，不必装 `rosbags`/ROS：

```python
import sqlite3
con = sqlite3.connect(f'file:{db3}?mode=ro', uri=True)   # 只读打开
rows = con.execute(
    'SELECT t.name, t.type, COUNT(m.id) '
    'FROM topics t LEFT JOIN messages m ON m.topic_id=t.id '
    'GROUP BY t.id').fetchall()
```

这一步扒出来：rosbag 里只有 `/svtrobot_cmd`(Twist)、`/f710/joy`、`/chassis/joint_states` 等
控制话题，**没有任何位姿** —— 这个结论决定了后面要自己推轨迹。

### 3.2 CDR 解码：把二进制 blob 还原成数值

`messages.data` 是用 **CDR**（ROS2 默认序列化格式）编码的二进制。对简单消息可以手工解。

`geometry_msgs/Twist` 的结构 = 两个 `Vector3` = 6 个 `float64`：
```
linear:  {x, y, z}
angular: {x, y, z}
```

CDR 编码 = **4 字节封装头 + 6 个小端 float64**，共 4+48 = 52 字节。验证：

```python
import struct
# data 是从 sqlite 取出的 52 字节 blob
vals = struct.unpack_from('<6d', data, 4)   # '<6d'=小端6个double, 从偏移4开始(跳过头)
v     = vals[0]   # linear.x  = 前进线速度 (m/s)
omega = vals[5]   # angular.z = 偏航角速度 (rad/s)
```

> `<` 小端、`6d` 六个 double、`unpack_from(..., 4)` 跳过 4 字节 CDR 头。
> 这就是「反序列化」最朴素的样子：知道字节布局，就能还原结构体。

### 3.3 ZED 点云 → BEV：俯视投影的本质

**BEV（Bird's-Eye-View）= 把 3D 点云从上往下拍扁成 2D 栅格图。**

ZED npz 里 `xyzrgba` 形状 `(720,1280,4)`，前 3 通道是每像素的 X/Y/Z（单位**毫米**）。
坐标系：**X=左右，Y=上下，Z=前方深度**（Z 全为正，因为相机看不到背后）。

最朴素的 BEV：把每个点的 (X, Z) 落到一个网格里，有点的格子标 1（占据）：

```python
xyz = np.load(f)['xyzrgba'][...,:3].reshape(-1,3) / 1000.0   # mm→m
xyz = xyz[np.isfinite(xyz).all(1)]                            # 去掉 NaN（无返回的像素）
ix = ((x - x_min) / RES).astype(int)    # 米 → 格子下标，RES=0.05 即 5cm/格
iz = ((z - z_min) / RES).astype(int)
grid[iz, ix] = 1
```

但直接这么投会**糊成一团**，因为有个陷阱 ↓

### 3.4 相机是斜的！—— 重力对齐（这步是精华）

**现象**：直接投影，地面会铺满整张图。
**诊断**：用 RANSAC 拟合点云里最大的平面（地面），看它的法向量：

```python
plane, inliers = pcd.segment_plane(distance_threshold=0.05, ransac_n=3, num_iterations=500)
# 结果法向 ≈ (0.03, 0.77, 0.64)，不是竖直的 (0,1,0)！
```

法向量 (0.03, 0.77, 0.64) 说明 **ZED 相机是俯视约 40° 安装的**。
相机斜着看，地面在 (X,Z) 平面投影里就被拉成一大片 → 糊。

**RANSAC 是什么**：随机采样一致性。反复随机取 3 个点定一个平面，数有多少点贴着它，
选「贴合点最多」的平面。对「一大片地面 + 零散障碍」的点云，它能稳健地把地面拟合出来，
不被障碍点干扰。这是点云处理的基本功。

**解法 —— 把点云摆正（重力对齐）**：求一个旋转矩阵，把地面法向量转到竖直 (0,1,0)，
整片点云跟着转，地面就水平了。用 **Rodrigues 旋转公式**（绕「两向量叉积」轴转「夹角」）：

```python
def rot_align(a, b=(0,1,0)):       # 求把单位向量 a 转到 b 的旋转矩阵
    a = a/np.linalg.norm(a); b = np.asarray(b,float)
    v = np.cross(a, b); s = np.linalg.norm(v); c = np.dot(a, b)
    vx = np.array([[0,-v[2],v[1]],[v[2],0,-v[0]],[-v[1],v[0],0]])   # 叉积的反对称矩阵
    return np.eye(3) + vx + vx@vx*((1-c)/(s*s))                      # 罗德里格斯公式
```

摆正后地面法向竖直、地面降到 Y=0，这时 (X,Z) 投影才是**几何正确的真俯视图**。

### 3.5 去地面：只留竖直结构

摆正后，地面在 Y≈0，墙/护栏/障碍是「离地有高度」的点。保留一个高度带就把地面滤掉了：

```python
mask = (y_aligned > 0.15) & (y_aligned < 1.6)   # 离地 0.15~1.6m 的点 = 墙和障碍
```

效果：占据率从 32%（全是地面）降到 5%（清爽的墙线）。出来的图能看出
「机器人在通道里，两侧各一道墙，间距约 6m」—— 对应任务表里的「直行通道(有护栏)」场景。

### 3.6 没有位姿，怎么还原轨迹？—— 航位推算（Dead Reckoning）

既然没有 GPS/SLAM 位姿，就用「**速度积分**」推位置。机器人每时刻有 (v, ω)：

```
θ(t+dt) = θ(t) + ω·dt          # 朝向 = 角速度积分
x(t+dt) = x(t) + v·cos(θ)·dt   # 位置 = 线速度沿当前朝向积分
y(t+dt) = y(t) + v·sin(θ)·dt
```

代码（v、ω 来自 3.2 解码的 `/svtrobot_cmd`，dt 来自相邻消息的纳秒时间戳之差）：

```python
dt = np.diff(ts, prepend=ts[0]); dt = np.clip(dt, 0, 0.5)   # 限幅防止大间隔跳变
x = y = th = 0.0
for i in range(len(ts)):
    th += w[i]*dt[i]
    x  += v[i]*np.cos(th)*dt[i]
    y  += v[i]*np.sin(th)*dt[i]
```

23940 条指令、942 秒，积分出一条覆盖 ~45×48m、总转角 1317° 的巡检环路。
**注意**：dead reckoning 会**累积漂移**（误差越积越大，终点对不齐），这是它的固有缺点，
正常现象；要消除得靠 SLAM 回环或轮速+IMU 融合，属后续工作。

### 3.7 没有显示器，怎么出动画？—— headless 渲染

服务器没有图形界面，所以**不能开 pygame 窗口**，要直接把动画写成文件。
用 matplotlib 的 `Agg` 后端（纯内存绘图）+ `FuncAnimation` + `PillowWriter` 导出 GIF：

```python
import matplotlib; matplotlib.use("Agg")          # 关键：无显示后端
from matplotlib.animation import FuncAnimation, PillowWriter
def update(i):                                     # 每帧更新箭头位置和朝向
    arrow.set_offsets([[xs[i], ys[i]]])
    arrow.set_UVC(np.cos(ths[i]), np.sin(ths[i]))
anim = FuncAnimation(fig, update, frames=240)
anim.save("traj.gif", writer=PillowWriter(fps=20))
```

箭头用 `quiver`（带方向的箭头），`set_UVC` 实时改朝向 = 展示 θ 的变化。
